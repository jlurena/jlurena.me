HealthNet Project Details

  • Summary
  • User Types and Use Cases
  • Development

HealthNet is a system that improves a hospital's workload by providing an easy mechanism for managing employees, gathering statistical data on the inner workings of the hospital, signing up patients, creating appointments, allowing ease of transfer of both patients and their information between hospitals and much more.

  • Patient
  • Doctor
  • Nurse
  • Hospital Administrator
  • Root/Admin
  • Can register him/herself through the registration page provided the needed information.
  • View his/her appointments in a monthly, weekly or daily calendar view.
  • Create appointments (Limited to selection of start time).
  • Cancel or edit appointments
  • View his/her prescriptions
  • View his/her, past and approved for release, test results.
  • A profile is created for the patient and the patient can edit when need to.
  • Print an electronic medical record.
  • Can view his/her appointments in a monthly, weekly or daily calendar view.
  • Can create, edit or cancel appointments with any of his/her patients.
  • Can create prescriptions for any of his/her patients and/or delete prescriptions.
  • Can publish medical tests and allow release of it for a patient to view.
  • Can view ANY patient's medical record.
  • Update any of his/her patient's medical profile.
  • Can admit or discharge ANY patient from a hospital. Can transfer any patient into one of his/her hospitals.

Nurses have limited access to patients belonging to the same hospital

  • Can update patient medical records.
  • Can create or modify appointments with any of the doctors and/or patients.
  • Can view all appointments for the week or day for ALL doctors and patients.
  • Can view prescriptions of all his/her patients.
  • Can admit a patient to his/her hospital.
  • Can register a doctor, nurse and hospital administrator.
  • Can view ALL appointments belonging to the same hospital.
  • Can view the system logs which include basic information like who logged in etc.
  • Can view compiled statistics such as which doctor has the most appointments etc.
  • Can transfer ANY patient into his/her hospital.
  • Can create, modify or delete any object of the system including users using the Django admin console.
  • Can upload a CSV file containing all objects to pre-seed or re-seed the system.

There are a total of five (5) users in this system and each of them have their own unique limitations and access throughout the entire system.
